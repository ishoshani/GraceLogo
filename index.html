Grace Logo Dialect Docs

Welcome to a beginners guide to using Logo in Grace!

This is a simple dialect that allows easy access to turtle graphics


We choose to keep things simple first of all by restricting the kind of variables e can declare(and we do have to declare them as a type)


Number like Var i : Number := 50

Boolean like Var i : Boolean := true

String like Var i : String := "Hello World"


Next we consider the attributes for our turtle


location a Point such as 250@250
heading a Number in degrees such as 270
nib a Boolean such as true for knowing if we are currently drawing

we control these using the following methods:


right(degree:Number) and left(degree:Number) turn us one way or the other
setAngle(degree:Number) lets us set an angle of our choice starting with right along the x-axis as 0

moveForward(units:Number) moves our turtle a number of units in its current direction, the heading
nibSwitch alternates whether our pen is on or off
setNib(isOn:Boolean)) lets us manually set the nib to On as true or Off as false

before we do anything, we want to start up the picture! So use show for that!

Make sure popups are allowed though, and that you have the objectdraw library present!


An Example to get us started!
--> 
dialect "logoDialect"
show // creates our 500 by 500 window, and starts the turtle at 250,250, with initial heading 270
right(90)
nibSwitch //turns on the nib, since it starts at off
moveForward(50)
nibSwitch
right(90)
moveForward(50)
left(90)
nibSwitch
MoveForward(50)

This Should give us

We also can use all the basic Control Structures!
--> 
dialect "logoDialect"
var i : Number := 0
show
while{(i < 4)}do{
    setNib(true)
    right(90)
   
    if(heading==90)then{
        setNib(false)
    }
    moveForward(50)
    i := i+1
}

This Should give us
-->

We are allowed to set location to any point, but in order to keep the angle system working we use a method for the heading
--> 
dialect "logoDialect"
show
var nx : Number := 50
var ny : Number := 150
location := nx@ny// we can set heading
nibSwitch

Developer Stuff

don't bother looking if you are just using the library

This library requires the use of lazy initialization. You can't use anything that you see at the top level! you must first use one of the lazyInit Methods for any element before you use them. This is to cover for in case the user forgets to start with show, or wants to set something before actually using the window. If you add new elements, especially any that involve the graphic view rather than the model, make sure you initialize the new elements at the top level as the notInitialized Object, then add a lazy initializer that checks if that notInitialized is equal to that Object. If they are equal, then you can initialized it! This is all necessary because top level code actually occurs while the compiling is happening, so a a graphics application can not appear from the browser during that time! We lazy initialize all our none boolean elements to stay consistent though.
