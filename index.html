Grace Logo Dialect Docs

Welcome to a beginners guide to using Logo in Grace!

This is a simple dialect that allows easy access to turtle graphics


We choose to keep things simple first of all by restricting the kind of variables e can declare(and we do have to declare them as a type)


Number like Var i : Number := 50

Boolean like Var i : Boolean := true

String like Var i : String := "Hello World"


Next we consider the attributes for our turtle


location a Point such as 250@250
heading a Number in degrees such as 270
nib a Boolean such as true for knowing if we are currently drawing

we control these using the following methods:


right(degree:Number) and left(degree:Number) turn us one way or the other
moveForward(units:Number) moves our turtle a number of units in its current direction, the heading

before we do anything, we want to start up the picture! So use show for that!

Make sure popups are allowed though, and that you have the objectdraw library present!


An Example to get us started!
--> 
dialect "logoDialect"
show // creates our 500 by 500 window, and starts the turtle at 250,250, with initial heading 0
right(90)
nib:=true //turns on the nib, since it starts at off
moveForward(50)
nib:=nib.not
right(90)
moveForward(50)
left(90)
nib:=nib.not
moveForward(50)

This Should give us
-->

Notice that (0,0) is at the top left, and becuase of that 90 degrees is actually down, while 0 degrees is still to the right.
We also can use all the basic Control Structures!
--> 
dialect "logoDialect"
var i : Number := 0
show
while{(i < 4)}do{
    nib:=(true)
    right(90)
   
    if(heading==90)then{
        nib:=(false)
    }
    moveForward(50)
    i := i+1
}

This Should give us
-->

We can also make methods to create shortcuts for procedures later. also note we can set location and heading using :=. Finally, notice that arguments can have type declarations, but don't necessarily have to.
--> 
dialect "logoDialect"
method DrawSquareSize(n:Number)TopLeft(place){
    location:=place
    heading := 0
    var i : Number := 0
    nib:=true
    while{i < 4}do{
        moveForward(n)
        right(90)
        i := i+1
    }
}
show
right(47)
nib:=true
moveForward (90)
DrawSquareSize(60)TopLeft(300@400)
}

This Should give us

Developer Stuff

don't bother looking if you are just using the library

This library requires the use of lazy initialization. You can't use anything that you see at the top level! If you want to expand it with a new field for the top level, espcially an object, you need to do it in steps. At the top level, create a private field(rememeber that fields are private by default in Grace), and add a "_" prefix to it in order to distinguish it as the private version. Then set that to the notInitialized object. Create a accessor method by using what you would want to name your field, and after checking if it is the the notInitialized object currently, assign it to your liking. if you want this field to be assignable, also create a ":=" method that just assigns the private field whatever is in the argument, which is is effectivly just the left side of an assigment operation for the user.

We also have the asRadian(degree),asDegree(radians) to convert our units, as well as asAngle to automatically set The degree of our heading within 360, with 0 point right along the x-axis and 90 pointing directly up.
